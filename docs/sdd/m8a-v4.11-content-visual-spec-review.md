# M8A-V4.11 Content + Visual Followup Spec Review

Date: 2026-05-05
Status: doc-only critique; no runtime, smoke, implementation spec, or code change

## §1 Spec Quality Verdict

The spec is directionally valuable because it stops pretending r2 was a full design pass. Its strongest move is naming that styling tokens did not fix the real product problem, then proposing per-window operator questions, summary-first evidence, disabled-module treatment, narrow viewport work, and a reviewer-mode pause concept.

The spec is not yet internally stable enough to hand to an implementer. It overstates some Phase 6 hookpoints, leaves Boundary ownership split across Decision/footer/status without a clear surface owner, conflicts with current V4.11 orbit color tokens, and treats reviewer mode as one bullet even though it changes replay state, inspector state, transition toast semantics, ARIA, and smoke expectations.

Overall verdict: **NEEDS-AMEND before implementation prompt**. It is a useful critique/spec draft, but it still has blockers that would force implementers to invent decisions mid-flight, especially around truth-boundary placement, Phase 6 hookpoint copy, orbit color migration, and pause/pin runtime behavior.

## §3 Critical Blockers

1. **Boundary ownership is not implementable as written.** §4.2, §4.4, and §4.6 conflict on whether Scale/Endpoint boundary belongs inside Decision content, above Decision, footer, or status. This is not pedantic because it determines DOM, tab count, default visibility, smoke assertions, and reviewer first-read behavior.
2. **Phase 6 hookpoint copy is not source-accurate.** "Phase 6 console", "comm-time", and "validation-boundary panel" need correction to actual accepted surfaces and current reachability before disabled tiles ship to customer.
3. **Reviewer mode lacks a replay state-machine contract.** The controller can pause replay, but Details open does not currently pause or pin state; toast/resume behavior is undefined.
4. **Orbit color semantics conflict with current V4.11 tokens.** The spec reverses current LEO/MEO/GEO color meaning without a migration plan.

## §5 Decision Recommendations On §12 Open Questions

1. **Reviewer mode default**: amend. Keep a visible Review toggle, but default it on for customer review/demo capture contexts and persist user choice; default-off alone weakens the very reviewer comprehension problem §7 is trying to solve.
2. **Inspector close on window change**: amend. Use strict pause-on-open and resume from pinned clock by default; do not let an underlying clock continue and then "catch up" unless a separate explicit control exists.
3. **Disabled module hookpoint text**: disagree. Current suggested copy should not ship. Use actual names such as `Communication Time`, `Handover Decision`, `Physical Input`, and `Validation State`, plus a reachability qualifier like "accepted Phase 6 bootstrap HUD surface" or "not reachable from this scene."
4. **Color semantics for W1/W2/W4**: amend. Same orbit family is acceptable only after resolving the current token conflict; W1/W2/W4 must still differ by role label, active state, sequence position, and motion/shape, not shade alone.
5. **Decision tab text language**: agree. Pure Chinese body copy with small English technical tags is the right balance for customer-facing comprehension and auditability.
6. **Validation boundary status corner**: amend. Do not put it into the already dense top strip unless a new right status slot is explicitly added. Prefer a low-prominence inspector-header badge when Details is open plus a footer/status chip when closed.
7. **Tablet 768x1024 design**: amend. Do not assume desktop layout with rail collapsed is enough; define portrait tablet as closer to narrow drawer/full-screen inspector and landscape tablet as a constrained desktop variant.
8. **Implementation phasing**: amend. Do not start implementation until blockers are resolved; then use at least four phases: content/hookpoint reconciliation, inspector/Boundary restructure, token/layout migration, reviewer-mode state machine + narrow viewport, followed by reconciliation captures/smokes.

Decision count: agreed 1/8, amended 6/8, disagreed 1/8.

## §6 What's Still Missing

1. A boundary ownership matrix: boundary subtype, owner surface, default visibility, interaction path, and smoke assertion.
2. Exact Phase 6 hookpoint labels and route/reachability behavior for accepted surfaces.
3. A complete customer must-have gap map covering policy switching, dynamic parameters, report export, communication rate, and real-time/prerecorded TLE mode.
4. Reviewer-mode state machine: pinned window/time, pause reason, resume policy, toast behavior, speed controls, final-hold behavior, and ARIA announcements.
5. Token migration plan for current `--m8a-v411-orbit-*` variables and screenshot/smoke impact.
6. Narrow/tablet interaction contract including drawer, modal, focus trap, `dvh`, safe-area, scroll lock, and return focus.
7. Disabled metric tile interaction/accessibility contract.
8. Metric tile typography contract: tabular figures, units, stable widths, placeholder states, and value hierarchy.
9. A concrete before/after content inventory for every Decision/Metrics/Evidence row by W1-W5.
10. Visual acceptance rubric against vc3d/vc3e reference screenshots, not just subjective 5/10 or 7.5/10 scoring.
