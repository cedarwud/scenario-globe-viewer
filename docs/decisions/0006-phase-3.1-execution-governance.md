# ADR 0006: Phase 3.1 Execution Governance

## Status

Accepted

## Context

Phase 2 implementation and Phase 2 close-out governance are both complete on the current `main` line, and the accepted evidence chain is already repo-owned: `npm run test:phase1` plus the Phase 2.12 visual baselines.

The missing Tier 1 / Tier 2 Profile A measurement evidence recorded in ADR `0005-perf-budget.md` still leaves the formal pre-Phase-3 measurement gate open. The current workspace is also still a WSL-backed development environment that supports build, smoke, and visual-baseline work but does not qualify as an admissible real-machine measurement path.

The repo nevertheless needs a stable handoff point for the finished Phase 2 baseline and a narrow way to begin the first Phase 3 shell slice without pretending the measurement gate has been closed.

## Decision

- Capture the accepted Phase 2 close-out state as a dedicated commit before any Phase 3.1 implementation change lands.
- Tag that commit as the Phase 2 completion anchor for later diffing, prompt handoff, and rollback.
- Allow only `3.1` (`feat(app): hud frame with empty panels`) to begin on top of that fixed baseline under WSL development conditions.
- Do not reinterpret WSL smoke runs, WSL visual refreshes, or SwiftShader browser passes as admissible Tier 1 / Tier 2 measurement evidence.
- Do not treat this exception as automatic authorization for `3.2+`, full Phase 3 close-out, or a later phase transition. Those require a fresh governance pass.

## Consequences

- The repo gains a clean Git handoff point for "Phase 2 complete" before the first Phase 3 implementation commit.
- The next implementation prompt can stay tightly scoped to the HUD-frame slice instead of reopening all of Phase 3.
- Formal Phase 3 readiness remains measurement-incomplete even after `3.1` starts, so later prompts and reviews must continue to distinguish "implementation allowed" from "gate closed."
- ADR `0005-perf-budget.md` remains the authority for the missing measurement evidence and accepted residual warnings; this ADR only governs the allowed execution scope while that evidence is still missing.
